Setting Timing

 

Step 4 of 6:

 

Often the effects of an event last for a period of time, gradually fade, or can be delayed.  In this scenario, the event “Caught wearing wire” only happens once, but it continues to provide motivation long after the actual event occurs.  However, we can consider that Tony might be just a little hotheaded, so maybe over time, the motivation will gradually decline.

 

To model this, right click on the arrow for the Motivation mechanism.  From the popup menu, select “timing” and the timing and scheduling dialog will appear.  In the top portion of the dialog, there are three fields in the box labeled “Timing.”  The first, Delay, affects how long after a cause occurs that the effect/signal will actually take place.  The second, Persistence, specifies if something lasts for some particular length of time.  The last one, Continuation, is the one we would like to set.  It will be a value between 0-1 specifying what proportion of the event/mechanisms probability will persist to the next timestep.  Here, let us set this value to 0.8.  This says that every timestep, the probability will be 80% of what it was the previous timestep.
