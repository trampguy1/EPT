Events

 

An Event is something that can occur in the real world that is also of importance to the situational model simulated by JCAT.  If there are any actions or occurrences that are of interest in the scenario being modeled, or if there are any actions or occurrences that may affect those Events in which we are interested, these actions should be included as Events in the Model.  Some examples of Events include:

 

• Bomb bridge

• Army X invades Country Y

• Enemy forces surrender

 

In JCAT, Events are modeled as processes which are caused (initiated) by signals and then, in turn, produce other signals (effects).  These processes can also be inhibited by incoming signals.  Events should be concrete, observable things.  Abstract ideas and motives should be represented as Mechanisms.

 

Tips:  

 

  • When creating a new Model from scratch, a good strategy is to first list the Events which you are mainly concerned, then move backwards and list the Causes or Inhibitors of those events.  For example, if your main concern is whether Country X will launch an air strike, list this as an Event first, and then list the Events affecting it: perhaps an invasion into Country X by Army Y could be a Cause, or loss of power at Country X’s air base could be an Inhibitor.

 

  • If an Event has multiple Causes or Inhibitors, make sure you set up any necessary group probabilities in the Specified Probabilities tab in the Event Editor dialog.
